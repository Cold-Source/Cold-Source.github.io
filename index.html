Правила контрибьютинга

1. Rebase-Only Workflow

Мы поддерживаем чистую линейную историю. Никаких «merge commits». Если вы новичок в git rebase, наши мейнтейнеры помогут вам подготовить PR.

2. Meaningful Commits

Мы используем Conventional Commits как стандарт для автоматической генерации changelog:

  • feat: — новый функционал
  • fix: — исправление ошибок
  • refactor: — оптимизация кода
  • docs: / test: — документация и тесты

Каждый PR проходит code review и автоматические проверки CI.
